# SkyPilot Heartbeat Telemetry Scripts

Scripts for analyzing and visualizing SkyPilot server heartbeat telemetry data (GPU allocation, Kueue queue health, borrowing ROI).

## Files

| File | Description |
|------|-------------|
| `analyze_heartbeat_roi.py` | Fetches heartbeat data from Loki (via Grafana proxy), computes per-server GPU allocation stats and Kueue borrowing ROI, and optionally generates plots and CSV reports. |
| `inject_test_heartbeats.py` | Injects realistic mock heartbeat data into Loki for testing. Generates data for multiple mock servers with varied GPU allocation patterns and Kueue borrowing dynamics. |
| `grafana_heartbeat_dashboard.json` | Importable Grafana dashboard for per-server heartbeat visualization. Generated by `/tmp/gen_dashboard.py`. |
| `heartbeat_schema_example.json` | Reference schema for the heartbeat JSON payload. |
| `heartbeat_example.json` | Example heartbeat payload. |

## Quick Start

### Analyze heartbeat data

```bash
python scripts/analyze_heartbeat_roi.py \
    --grafana-url http://<grafana-host>:3225 \
    --grafana-user admin --grafana-password <password> \
    --range 7d --plot-dir ./roi_plots --csv-output report.csv
```

### Inject test data into Loki

```bash
python scripts/inject_test_heartbeats.py \
    --loki-url http://<loki-host>:9090 \
    --duration 24h --interval 600 --dry-run
```

### Import the Grafana dashboard

1. In Grafana, go to **Dashboards → Import**
2. Upload `grafana_heartbeat_dashboard.json`
3. Select your Loki datasource when prompted

The `server_hash` dropdown shows labeled entries (e.g., "CustomerA (7d420045)"). To update labels: **Dashboard Settings → Variables → server_hash → edit "Custom options"** — no JSON re-import needed. You can also type arbitrary hashes directly in the dropdown.
